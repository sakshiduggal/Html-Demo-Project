HTML Elements

  • The Root Element

    1. The html element represents the root of an HTML document.

  • Document Metadata

    1. The head element represents a collection of metadata for the Document.

    2. The title element represents the document's title or name. Authors should use titles that identify their documents

      There must be no more than one title element per document.

    3. The base element allows authors to specify the document base URL for the purposes of resolving relative URLs

    4. The link element allows authors to link their document to other resources.

      The destination of the link(s) is given by the href attribute. A link element must have rel attribute.

    5. The meta element represents various kinds of metadata that cannot be expressed using the title, base, link, style, and script elements.

    6. The style element allows authors to embed style information in their documents. The style element is one of several inputs to the styling processing model. The element does not represent content for the user.

  • Scripting

    Scripts allow authors to add interactivity to their documents.

    1. The script element allows authors to include dynamic script and data blocks in their documents. The element does not represent content for the user.

    2. The noscript element represents nothing if scripting is enabled, and represents its children if scripting is disabled. It is used to present different markup to user agents that support scripting and those that don't support scripting, by affecting how the document is parsed.

  • Sections

    1. The body element represents the main content of the document.

    2. The section element represents a generic section of a document or application. A section, in this context, is a thematic grouping of content, typically with a heading.

    3. The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links.

    4. The article element represents a self-contained composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.

    5. The aside element represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.

    6. The elements h1 h2 h3 h4 h5 h6 represent headings for their sections.

      The semantics and meaning of these elements are defined in the section on headings and sections.

      These elements have a rank given by the number in their name. The h1 element is said to have the highest rank, the h6 element has the lowest rank, and two elements with the same name have equal rank.

  • Grouping content

    1. The p element represents a paragraph.

      The p element should not be used when a more specific element is more appropriate like address element.
